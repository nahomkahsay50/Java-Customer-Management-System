# Java Customer Management System

A Java console application that reads, displays, and transforms customer data stored in XML format. Demonstrates XML parsing with the DOM API, XML file generation, structured logging, and clean separation of concerns across multiple classes.

---

## Overview

This program gives users a menu-driven interface to either view all customer records from an XML file or generate a new enriched XML file with additional contact fields. All view operations are tracked using Java's built-in `Logger`, which writes structured logs to an XML log file.

---

## Files

### Source (`src/week15finalproject/`)

| File | Description |
|---|---|
| `GetUserInput.java` | Entry point; displays menu and routes user choices |
| `ViewCustomers.java` | Parses `customers.xml` and prints a formatted customer report |
| `ModifyCustomers.java` | Reads `customers.xml` and generates an enriched `customer_modified.xml` |
| `package-info.java` | Package declaration for `week15finalproject` |

### Data

| File | Description |
|---|---|
| `customers.xml` | Source customer data (5 records with ID, name, type, address) |
| `customer_modified.xml` | Output file generated by Option 2 — includes added phone, contact name, and email fields |
| `customer_operations.xml` | XML log file auto-generated by the logger during Option 1 |

## How It Works

### Menu Options

```
Customer Management System
-----------------------------

Please select an option
1 - View Customers
2 - Create Modified Customers File
Your choice:
```

### Option 1 — View Customers

`ViewCustomers.DisplayCustomers()` uses the **Java DOM XML parser** to read `customers.xml` and print a formatted listing of all customer records.

**Logging:** Every significant operation is logged to `customer_operations.xml` using `java.util.logging`:

| Level | When it fires |
|---|---|
| `INFO` | Operation start, file found, customer count, completion |
| `WARNING` | No customers found in the XML file |
| `SEVERE` | File not found, XML parse error, I/O error |

### Option 2 — Create Modified Customer File

`ModifyCustomers.CreateModifiedCustomerFile()` reads `customers.xml` and appends three new fields to each customer, then writes the result to `customer_modified.xml`:

| New Field | Example Value |
|---|---|
| `phone` | `(513) 555-1000` |
| `contactName` | `John Smith` |
| `email` | `customer1@gmail.com` |

## Requirements

- **Java 8+**
- No external libraries — uses only the Java standard library (`javax.xml`, `org.w3c.dom`, `java.util.logging`)

## Running the Project

### In Eclipse
1. Clone the repository
2. **File → Import → Existing Projects into Workspace**
3. Right-click `GetUserInput.java` → **Run As → Java Application**

### From the Command Line
```bash
javac -d bin src/week15finalproject/*.java
java -cp bin week15finalproject.GetUserInput
```

> `customers.xml` must be present in the working directory when the program runs.
